# VCE English Teaching

A structured working repository for VCE English lesson planning, assessment design, exemplars, feedback systems, slide creation, and reusable AI prompts.

## Purpose

This repository is designed to streamline the planning and delivery of VCE English. It brings lesson materials, assessment resources, teaching prompts, slide outlines, and exemplar-based support into one organised workspace so units can be reused, refined, and adapted over time.

The aim is to reduce repeated admin work, improve consistency across classes, and create a clear system for teaching, feedback, and presentation design.

## What this repository includes

This repository may include:

- Unit plans for VCE English Units 1–4.
- Individual lesson plans and weekly teaching sequences.
- Assessment tasks and SAC preparation resources.
- Essay prompts and text response materials.
- Argument analysis and comparative analysis resources.
- Model paragraphs, exemplars, and annotation samples.
- Feedback banks and rubric-aligned comment frameworks.
- Claude prompts for lesson planning, feedback, and resource creation.
- Slide outlines and presentation materials for classroom teaching.
- Revision resources, checklists, and student support documents.

## How to use this repository

### 1. Lesson planning
Use the `Lesson-Plans` and `Units` folders to build or adapt teaching sequences. Each lesson should ideally include:
- Lesson objective.
- Success criteria.
- Key knowledge and skills.
- Learning activities.
- Discussion questions.
- Exit ticket or reflection task.

### 2. Assessment preparation
Use the `Assessment` folder for SACs, practice tasks, rubrics, and exam-preparation materials. This helps keep task design and marking criteria consistent across classes.

### 3. Text response teaching
Use the `Text-Response` folder for prompts, model paragraphs, essay structures, and annotated exemplars. These materials can support both explicit teaching and revision.

### 4. Argument analysis teaching
Use the `Argument-Analysis` folder for strategy-based analysis, language analysis, image analysis, and mentor responses. This section is especially useful for teaching students how to connect contention, argument, tone, strategy, and technique.

### 5. Feedback and marking
Use the `Feedback` folder to store reusable comment banks, rubric-linked feedback, and student target statements. This is the best place for storing feedback systems that can be reused across multiple classes and tasks.

### 6. Slide creation
Use the `Slides` and `Prompts` folders together. Build a lesson or workshop in outline form first, then turn it into presentation slides using a reusable Claude or Canva prompt.

## Recommended workflow

A practical weekly workflow for this repository:

1. Start in `Units/` or `Lesson-Plans/` and identify the lesson focus.
2. Draft the lesson sequence in a lesson plan template.
3. Pull supporting materials from `Text-Response/`, `Argument-Analysis/`, or `Comparative/`.
4. Use a saved Claude prompt from `Prompts/Claude/` to generate or refine activities, explanations, or model responses.
5. Save any classroom deck in `Slides/`.
6. Store strong exemplars, teacher notes, and successful revisions back into the relevant folder for future reuse.

## AI use

This repository is intended to work well with AI-assisted teaching workflows. Suggested uses include:
- Generating first-draft lesson plans.
- Creating slide outlines.
- Producing model paragraphs.
- Turning notes into revision resources.
- Drafting rubric-based feedback.
- Building differentiated tasks for students at different levels.

AI outputs should always be reviewed and refined before classroom or assessment use.
